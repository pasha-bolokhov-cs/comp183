The C Language

An Abridged Introduction

  • Designed by Dennis Ritchie c1972

  • Derived from B: in turn a modified form of Richards' BCPL (derived from CPL)

  • Used as the (re)implementation language for Unix

  • Achieved great popularity with the advent of the PC

    • Compilers relatively easy to write

    • Use relatively little memory

    • Produce efficient code

    • A vast improvement on both MASM and BASIC as an implementation language

  • Data types

    • Base types:

      • int integer

          • short [int]

          long [int]

          unsigned

        • C standard says nothing about size of short, int, or long, merely that:

          sizeof (short) <= sizeof (int) <= sizeof (long)

      • float real/floating point (single-precision)

        • float single precision
        • double double precision

      • char character
        • (C usually considers this as a short int)

    • Constants:

      • int 
        
int-const         ::= decimal-int | octal-int   | hex-int
decimal-int       ::= decimal-digit {decimal-digit}
octal-int         ::= 0 octal-digit {octal-digit}
hex-int           ::= 0x hexadecimal-digit
	              {hexadecimal-digit}
octal-digit       ::= 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7
decimal-digit     ::= octal-digit | 8 | 9 
hexadecimal-digit ::= decimal-digit | a | b | c
		      | d | e | f | A | B | C | D
		      | E | F
        Examples: 
        
1       decimal
12      decimal
012     octal (base 8)
0x1f    hexadecimal (base 16)
      • char 
        
char-const       ::= ' char '
char             ::= escape-char | octal-char 
                       | hexadecimal-char | regular-char
escape-char      ::= \ esc-special
esc-special      ::= a | f | n | r | t | \ | ' | "
octal-char       ::= \ octal-digit 
                       [octal-digit [octal-digit]]
hexadecimal-char ::= \x hexadecimal-digit 
                       [hexadecimal-digit]
regular-char     ::= None of the above.....
                       (i.e. a printing, typable character)

        Examples: 

        
'c'     the character c 
'\\'    the character \
'\''    the character '
'\t'    the tab (HT) character
'\n'    the newline (NL) character (or CR/LF)
'\071'  the character 9
'\x41'  the character A

  • Programs (simplified) 

      • 
program         ::= function {function}
function        ::= [return-type] identifier
                      '(' [parameter-list] ')' statement-block
return-type     ::= type
identifier      ::= A valid C identifier 
                      (implementation dependent)
parameter-list  ::= parameter { , parameter }
parameter       ::= type identifier
statement-block ::= '{' {declaration ;} {statement } '}'
declaration     ::= type identifier { , identifier }
statement       ::= simple-statement ; | statement-block
simple-statement::= expression | if | while | for 
                      | do | switch | ...
expression      ::= expression operator expression 
                      | function-call
operator        ::= = | ==| !=| < | > | <= | >= | + 
                      | - | * | / | % 
                      | | | || | & | && | ...
function-call   ::= identifier 
                      ( [ expression { , expression} ] )
if              ::=   if '(' expression ')'statement
                    | if '('  expression ')' statement
                        else statement
while           ::= while '(' expression ')' statement
for             ::= for '(' [expression] ; [expression] 
                      ; [expression] ')' statement
do              ::= do statement while '(' expression ')'
switch          ::= switch '(' expression ')'case-list
case-list       ::= case-statement 
                      | { case-statement {case-statement}}
case-statement  ::= case constant : statement-list 
                      | default : statement-list
statement-list  ::= {statement}
      A simple example 
      
#include <stdio.h>  /* This is a preprocessor directive,
                     not part of C */
#include <string.h> /* So is this */

/*
 * main is the compulsory name of the primary procedure
 * argc is an integer,
 * argv a pointer to a pointer
 * to character
 */

int main (int argc, char **argv){
  int i;                /* a single integer variable */

  printf ("Arguments in their original form, and lower cased\n");

  for (i = 1; i  < argc ; i++){
                        /* Iterate up command line arguments */
	printf ("argument %d = \"%s\"\n", i, argv[i]);
	printf ("argument %d (lowercase) = \"%s\"\n", i,
	        strlwr (argv[i]));
  }

  printf ("Th..th..that\'s all folks!\n");

}

    • Function prototypes

      • Used to ensure that

        • Function declarations agree with their specification (cf. DEFINITION and IMPLEMENTATION modules in Modula-2)

          • Functions may be used before they are defined providing a prototype has been specified (forward referencing)

        • Functions are used according to their specification

      • Specification: 
        • 
prototype   ::= return-type identifier '(' [ptype-list] [ , ...]')';
return-type ::= type
ptype-list  ::= type { , type}

        Examples: 

      
        int dummy ();

                /* A function returning an integer
                   params unspecified not recommended*/

        int dummy1 (int, char, int []);
                /* A function returning an integer,
                   taking 3 parameters: an int, a char and 
                   an array of ints */

        void dummy2 (void);
                /* A function returning nothing and taking no parameters */

        int dummy3 (char, int, ...);
                /* A function returning an integer,
                   taking a character parameter, an integer parameter, 
                   and then any number of parameters of any type */
  • The C preprocessor

    • A pre-compilation pass over the source

      Essentially rewrites the source file using definitions, macros, included external files

    • Output passed to the compilation phase

    • Basic syntax: 
      
cpp-directive ::= # directive
directive     ::= include | define | ...
include       ::= include include-file
include-file  ::= '<' filespec '>' | " filespec "
define        ::= define name definition
    • Header files
      • A header file is a file containing common definitions, functions prototypes, etc. usually useful for several other program

      • Roughly equivalent to a DEFINITION MODULE in Modula-2 although more flexible (i.e. easier to screw up)

      • Header files are included in the compilation process via the #include preprocessor directive
    • Defines (macros)
        • The only way in C to define a constant 
        
#define GST 7
#define MYNAME "Deryk"
        But beware: a #define is literally shorthand 

        
#define FOUR 2 + 2 
#define SIXTEEN FOUR * FOUR

/*
 * Does not work, because it is effectively:
 */

#define SIXTEEN 2 + 2 * 2 + 2

  • Arrays and strings

    • Array declarations

      • 
array-decl  ::= type array-ident { , array-ident }
array-ident ::= identifier '[' size ']' 
                  { '['size ']'}
      Examples: 
        int intarray[10]; 
        char string1[100], string2[20]; 
        float floatarray[10][20];
        Note:

          Array indices are from 0 to size - 1

    • Strings

        • A string is not a builtin type in C, it is a kludge based on an array of characters

        A string is represented as a sequence of bytes with a terminating \0 (i.e. NULL) character which identifies the end of the string. (This is what is often called an ASCIIZ string)

        Consequently, to hold a string of 20 characters an array of 21 chars is necessary 

            char string[21];

        All string manipulation is done with functions, whose prototypes are declared in the header file string.h, the only exception is initialisation:

        Example: 

        
#include <string.h>

void dummy (){
  char str1[100] = "This is a string", str2[100];

  strcpy (str1, str2);
                /* copy str2 to str1 */
}

  • Pointers 

      • pointer-decl ::= datatype * identifier
      Examples: 
      int *intp; /* intp is of type 
              pointer to int */

char *charp1, **charp2;
          /* charp1 is type pointer to char, 
             charp2 is type pointer to pointer
             to char */

    • Referencing and dereferencing:

      • Referencing 
        pointer-ref ::= & identifier
        Example: 
        p = &i; /* Assign to p the address 
	   of variable i */
      • Dereferencing 
        pointer-ref ::= * identifier
        Example: 
        i = *p; /* Assign to i the contents of the variable
           pointed to by p */
*p = i  /* Assign the contents of i to the variable
           pointed to by p */

    • Pointers and arrays

        • Pointers and arrays in C are intimately related 
        
int i, array[100], *p; 

p = array; 	/* Equivalent to p = &array[0] */
p++; 		/* p now contains &array[1] */
i = *p; 	/* Assign to i the contents of array[1] */
i = *(p+1);	/* Assign to i the contents of array[1+1]
		   i.e. array[2] */
        When a formal parameter of a function is either an array or a struct (see below) C will automatically force the actual argument passed to be a pointer Arrays and Pointers in C the difference between an array and a pointer is blurry, to say the least

  • Function invocation and parameter passing

      • All arguments to C functions are passed by value (as opposed to by reference or other mechanisms to be discussed later.

      In other words the called function receives a copy of the parameter.

      Example: 

      void badswap (int a, b){
  int z;

    z = a;
    a = b;
    b = z
}

badswap (x, y);

      badswap will not exchange the values of its arguments.

      Example: 
      void goodswap (int *a, *b){
  int z;

    z = *a;
    *a = *b; /* a = b is wrong - why ? */
    *b = z;

}

goodswap (&x, &y);

  • Structures

      • struct-decl ::= struct [identifier ] '{' member-list '}'
member-list ::= declaration ; { declaration ;}
      Examples: 
      struct test{int a, b; float z;};
struct test s1, s2;

  s1.a = 2; 
  s2.z = 3.1415926


struct testa{
  int x, y, z;
  float a, b;


  struct{
    char p, q, r;
  } internal;
} TA1, TA2;

  TA1.x = 5;
  TA2.b = 2.71828;
  TA1.internal.p = 'a';

  • Unions

      • union-decl ::= union [identifier ] '{' member-list '}'
member-list ::= declaration ; { declaration ;}
      Examples: 
      union uniontest {int a; float b; char c;};
union uniontest u1, u2;
int x;
float y;

  u1.a = 5;
  u2.b = 1.4142;
  x = u2.a; /* What is the value of x? */
  y = u1.b; /* What is the value of y? Who knows? */
