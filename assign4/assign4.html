Instructions

Lab 4

For demonstration during week 8

20 Marks

(Work in pairs)

Topics

  • Passing and retrieving parameters on the stack
  • Multiplication

Assignment

You are to write a program which includes a procedure to multiply two numbers together and return their product. Code to input and output the numbers and convert from ASCII to binary etc is provided (see below).

  1. The driver program pushes both numbers and the address of the result onto the stack before calling the multiply routine
  2. If the product will not fit into 16 bits the product is invalid (you don't even need to store it) and the Carry flag will be set (we can discuss this in class).
  3. Your code must save and restore any registers it uses. The driver code checks this...

Notes

Your program should look, in outline, something like this:

	program	lab4
;
; Usual author name, comments etc.
;
	include	'/home/batemann/labs/183/Driver'

	data

;
; Any data you need
;

	code

lab4
	call	Driver

	sys	EXIT

;
; Code for multiply routine goes here - it will be called by the driver
;

multiply
	.....

	END

The Driver routine prompts for numbers, converts them to binary, pushes the parameters into the stack and then calls multiply.

Multiply should work as described above. It should not attempt any input or output.

It is a very bad idea to pop the nubers you're multiplying off of the stack.  Just don't do it.

EQUates for BIOS definitions (EXIT, CONSOLE etc), for handy characters (CR, NL, NUL), and for the individual bits in the word (Bit0 is the left bit, Bit15 the right bit) are already included and you should not redefine them.

You are also not allowed to use any of the labels or memory (other than the stack) that is defined in the driver program - I will check, you will loose points.
