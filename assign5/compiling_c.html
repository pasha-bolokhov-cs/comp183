Compiling and running C programs on DeepBlue

The compiler on DeepBlue is called gcc (the GNU C Compiler) and must be run from the command line.

In order to compile a program called, say test.c and call the resulting executable testuse the folling command:

	gcc -o test test.c

The -o specifies the name of the executable. If you leave this out, gcc will automatically save the executable as a.out(Don't ask)

To run the program:

  • If the current directory (.) is in your search path (which it is not by default - type echo $PATHto find out) then simply type the program's name: 
    	test

  • If the current directory is not in your search path, then the program's path. The simplest way is: 
    	./test

  • To debug the program:

    There is no nice function-key driven debugger on Unix systems. Remember "printf is your friend"!

    As you recall from COMP 176, there is a debugger available called gdb. In order to use this, remeber; you must also specify the -goption to gcc or it won't include debugging info in the exectutable.

    Remember:  "man" is your friend. 

There is a simple sample program (targs.c) attached to the bottom of this page, which shows a number of things; including how to access command line paramters and how to use printf - the formatted output function provided by C

You can get the documentation on printf online by typing:

	man 3 printf

And yes, as I pointed out in class; the 3 is important.
