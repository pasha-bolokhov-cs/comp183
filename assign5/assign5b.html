Instructions

Lab 5b (not so simples)

30 Marks

Topics

  • Trees and recursion
  • Expression evaluation

Before you start

Here is some information on compiling and linking C programs on DeepBlue.

Introduction

The evaluation of expressions is a central issue in the implementation of computer languages.

There are two basic approaches: syntax-directed and operator-precedence parsing. This lab deals with the first of the two.

Assignment

Lab 6b consists of several C source files, all of which are linked together to form a small interactive expression interpreter/evaluator.

This language supports integer-only arithmetic (including division). The operators which you should implement are:

  • + (addition)
  • - (subtraction)
  • * (multiplication)
  • / (division - integer-only remember)
  • % (modulus - remainder)

These are the files; you will find them all linked at the bottom of this page except for machine.c, which you must write. This is your assignment.

  • lab5b.c, the main program, it contains what in LISP interpreters is called the read-evaluate-print loop. The main function calls:
    • Parse, contained in parser.c, which inputs an expression, checks its syntax and, if it is well-formed, builds an expression tree.
    • PrintParseTree, contained in machine.c, which prints out the expression tree. You write this.
    • PrintPostTree, contained in machine.c, which prints out the postfix version of the expression tree. You write this.
    • EvaluateParseTree, contained in machine.c which evaluates the expression tree and returns the calculated value. You write this too.
  • lex.c, contains the code to read from the input stream (stdin) and assemble it into the basic tokens of the expression language.
  • parser.c, contains the code to analyse the input expression, check it for errors and build an expression tree.
  • error.c, contains simple error-handling code.
  • tree.c, which contains the functions for making nodes and leaves in the expression tree.

There are also a number of include files, which you need. They are in the same directory. They are:

  • global.h, contains common definitions used throughout the source code.
  • machine.h, contains the prototypes for the functions contained in machine.c, which should #include both this and global.h
  • error.h
  • lex.h
  • parser.h
  • tree.h

The last four are all used by the respective .c files.

if you save and unzip "Makefile" from Makefile.zip below, then you can simply type

make

instead of gcc'ing every source file individually.

You can check out the interaction by running either the 32-bit binary (from lab5b-32.zip below), or the 64-bit binary (from lab5b-64.zip below), depending on whether you're running 32-bit (including deepblue) or 64-bit linux.

Hints

The tree consists of nodes, each node is either a leaf, in which case it contains a number, or it is an application node, containing three things:

  1. the operator type (stored as a single char/int value: '+', '-', '*', '/' or '%')
  2. the pointer to the left subtree
  3. the pointer to the right subtree

Both the functions you must write recursively walk the tree.

  • PrintParseTree

    An in-order traversal of the tree; at each node:

    • If it is a leaf node, print the number contained in the node
    • If it is a non-leaf node, print a left parenthesis, print the left subtree, print the operator, print the right subtree, print a right parenthesis.

      A subtle programmer will arrange for the parentheses not to be printed at the top level. In other words if I type in the expression:

        5 + 6 * 4;

      The expression tree output should be:

        5+(6*4)

      and not:

        (5+(6*4))
  • PrintPostTree

    A post-order traversal of the tree; at each node:

    • If it is a leaf node, print the number contained in the node
    • If it is a non-leaf node, print the left subtree, print the right subtree, print the operator.

      It would be nice to have the entire postfix expression on one line.

  • EvaluateParseTree

    Also an in-order traversal. At each node:

    • If it is a leaf node, return the number contained in the node
    • If it is a non-leaf node, recursively evaluate the left subtree, then the right subtree, then apply the operator to the two values. Return the result.
