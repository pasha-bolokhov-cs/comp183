Instructions

Lab 2

To be demonstrated in the lab during week 4 or 5.

15 Marks

Topics

  • Simple Input & Output
  • Indexed addressing

Assignment

You are to write a small CHASM program which will:

  1. prompt the user for a string of up to 80 characters
  2. input the string
  3. exit if the input consists merely of the CR (carriage return) character
  4. print the string backwards
  5. repeat steps 1 to 4 until the lonely CR is entered.

Hints

You will need to use the CONSOLE BIOS functions, in particular GETSTR to read the input and PUTCHAR to output a single character (there is no "put string to screen backwards" function)

The input length (provided by GETSTR in byte buffer+1) will be zero if the user merely entered the CR.

You can easily find the last character in the input string by adding the length to the address of the first byte (and then subtracting 1 - why?)

You can move along to the previous byte in the string by subtracting 1 from the current index register.

Hand in

Again; just your source code.  You will earn most of your marks by demonstrating your working lab and explaining a bit about your program using the list file.

Important:  To avoid loosing marks your source code must be named "lab2.csm", be appropriately commented, and must contain at a minimum the following in your comments:

  1. Your full name and student ID number ("C"-number)
  2. The date and time you demonstrated your lab
  3. The unique "code word" that I supply you after you've demonstrated your lab.

    This version uses "fputs()" instead of "puts()" for printing the prompt

    The reason is that "puts()" automatically adds the new line ('\n') regardless of what is in the string.

    In contrast, "fputs()" only prints whatever it is asked
